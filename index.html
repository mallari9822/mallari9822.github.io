2. Ticketing System | Web Application

Technologies: NodeJS | Date: 2025

Description: This project is a robust, modular web application designed to manage event listings, user registrations, and administrative tasks. The system focuses heavily on backend architecture, utilizing a comprehensive set of API routes to handle everything from viewing events to confirmed registrations. The system is architecturally notable for its multi-layered security model, which accommodates public access, general user registration, specialized presale access, and secure administrator control, with JSON Web Tokens (JWT) as the primary authentication mechanism for users.

Key Features:

  • JWT Authentication: Users are authenticated using secure, stateless JSON Web Tokens (JWT) stored in httpOnly cookies.
  • Career Application Module: Dedicated public routes for displaying the career form and securely submitting job applications with resume uploads.
  • Multi-Role Access Control: Clearly defined and protected API routes govern four distinct user tiers: Public, Authenticated User, Presale Invitees, and Administrator.
Reflection: The heart of this project was the challenge of building a secure, seamless user experience in the backend. I learned that security isn't just about locks; it's about architecture. Mastering JSON Web Tokens (JWT) was the key—it allowed me to create a stateless, scalable system that could handle thousands of user logins without bogging down the server with sessions. Implementing the JWT flow, from secure creation to validation via httpOnly cookies, solidified my understanding of modern, distributed authentication. This focused work on the JWT security model and the subsequent design of complex, multi-role API endpoints showcases my commitment to building high-performing, user-focused, and robust backend services.

3. Dementia Data Builder (In-Progress) | Mobile Application

Technologies: Flutter, Dart | Date: 2025

Description: This is a mobile application developed to facilitate dementia research by collecting cognitive assessment data from participants. The app is built using Flutter and Dart. The application administers standard, clinically validated cognitive assessments, including tasks that measure scores, reaction times, accuracy, and audio recording capabilities. It is strictly a research tool designed for secure, anonymous data collection.

Key Features:

  • Research Focus: Designed exclusively to collect quantitative and qualitative cognitive assessment data for dementia research purposes.
  • Ethical Data Privacy: Ensures participant trust by collecting no Personally Identifiable Information (PII). All collected data is anonymized, stored securely, and used solely for aggregated, de-identified research analysis.
  • Voluntary Participation: Adheres to research ethics by allowing participants to withdraw at any time or skip any test without penalty.
Reflection: Developing this application in Flutter and Dart allowed for cross-platform deployment, which was essential for reaching a broad research participant base quickly. This project showcases my ability to develop highly functional applications that adhere to strict ethical and data privacy standards. This project is currently in progress, demonstrating my commitment to continuous development, security hardening, and iterative feature integration within a rigorous research framework.
